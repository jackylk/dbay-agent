# KB Schema

本文档告诉 wiki agent 如何维护这个知识库。Agent 在每次 ingest / curate / lint / chat 运行开始时读取本文档。你可以编辑它，agent 也会在工作中协同进化它。

## 范围 (Scope)

描述这个知识库的主题和边界。Agent 会据此判断新文档是否属于这里、某个概念是否值得单独建页。

> 请在这里填写你的知识库范围，例如：
> "本 KB 专注于 [主题]，包括 [子领域]，不包括 [排除项]。"

## 页面规范 (Page Conventions)

- **语言**: 所有页面使用简体中文，无论源文档语言。
- **页面大小**: 每页 800-2500 字。超过 3000 字应拆分。
- **标题风格**: 清晰的中文名词短语。避免通用标题如"概述"、"介绍"、"总结"。
- **文件名**: kebab-case，与概念对应（如 `注意力机制.md`）。
- **交叉引用**: 使用 `[[页面标题]]` 内链引用相关页面，标题必须精确匹配已有页面。
- **来源标注**: 引用具体事实时用脚注 `[^1]: 源文件名, 章节`。

## 创建 vs 更新预算 (Create vs Update Budget)

- **强烈优先更新已有页面**，而非创建新页面。有疑问时先搜索、再更新。
- **每次 ingest 目标**: touch 5-15 页（含更新）；创建新页面不超过 2-3 页。
- **仅在以下情况创建新页**: 概念是本 KB 领域的核心，且没有已有页面覆盖它，且它能回答一个独立的用户问题。
- **不要为以下内容建页**: 广为人知的通用概念、离题的旁支、可以融入已有页面的零散事实。

## 日志 (Log)

每次 ingest / curate / lint 运行结束时，通过 `log_note` 工具向 `log.md` 追加一行。格式: `[YYYY-MM-DD HH:MM] <操作类型> | <简短描述>`。

## 自我维护 (Self-maintenance)

当你发现本 schema 有错误、过时或缺少能改善工作质量的规则时，调用 `update_page` 修改 `schema.md`。
