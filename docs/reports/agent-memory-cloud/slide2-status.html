现状：Agent 记忆的三个阶段

从纯文本到语义检索，再到云端服务

Claude Code

纯 Markdown 文件

  • MEMORY.md + 独立记忆文件
  • 检索方式：grep / 文件名匹配
  • 200行截断，需人工修剪
  • 可 git 同步，但无语义理解
  • 单人单项目，跨项目靠手动

类比：记事本

OpenClaw

Markdown + SQLite 向量

  • Markdown存储 + sqlite-vec 向量索引
  • 混合检索：70%语义 + 30%BM25
  • 本地 .sqlite 文件，不可网络共享
  • SQLite 单进程，无并发写入
  • 索引是二进制，无法 git 同步

类比：个人笔记软件

Cloud Memory

云端记忆服务 (DBay)

  • 云端存储 + 向量 + 全文 + 结构化
  • API 接入，任何 Agent 可调用
  • 多用户、多项目、多Agent共享
  • 租户隔离 + 权限控制 + 审计日志
  • 记忆属于用户，不属于 Agent

类比：企业知识平台

CC 的 Markdown 能 git 同步但无语义 · OpenClaw 有语义但 SQLite 锁死单机 · 两者都困在本地范式
